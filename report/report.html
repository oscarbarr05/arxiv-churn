arXiv Researcher Churn Predictor

Predict who stops publishing · explain why through feature selection, PCA/SVD & co-authorship structure · act with paper recommendations.

Final Report — Introduction to Data Science, UPTP (Units 8–9) · Oscar Barrios · github.com/oscarbarr05/arxiv-churn

Headline. 295 cs.CL researchers, churn rate 18.6%, leakage-free temporal label. Consensus of four selection methods → 8 features. RandomForest 5-fold CV F1 0.65 / accuracy 88%, rising to F1 0.68 with co-authorship centralities. SVD recommender (84% variance) gated at churn prob ≥ 0.5. Fully Dockerized: docker-compose up --build → API + interactive dashboard, no API key, no manual steps.

1 · Problem & data

Like Netflix/Spotify retention systems, the pipeline does three jobs: predict churn, explain it, and act on it. The entity is an author in arXiv's cs.CL category; data comes from the public arXiv API (no key). A two-stage scraper first samples a frame (100 papers/quarter, 2014–2025 ≈ 4 800 papers — sampling all years avoids a recency-biased cohort), then fetches each discovered researcher's full history (20 333 author-paper rows), rate-limited at 3 s/request with defensive parsing and pagination.

Churn label (leakage-free). A researcher has churned if they submitted no paper after the cutoff 2024-06-30 (≈2 yr silence). Features are computed only from pre-cutoff papers and the label only from post-cutoff activity — so recency is a legal predictor instead of leaking into the label. Classes are balanced (18.6% churn, above the 10% danger zone); all models also use class_weight="balanced".

No raw API field is used directly. 14 features span four families covering recency, frequency and magnitude (full definitions in the repo):

3 · Feature selection — four methods + consensus

Filter (variance + correlation pruning |r|>0.9 + ANOVA), Wrapper (RFE/logistic, top-5), and Embedded (DecisionTree max_depth=5, RandomForest n_estimators=100). A feature is kept on ≥2 of 4 votes; the rule is shared by notebook and API so they never drift.

Disagreements are the insight. papers_per_year is near-last for ANOVA (which scores features in isolation) yet top-5 for RFE and both trees — its signal is interaction-only. is_multidisciplinary is the reverse (high ANOVA, last RF) — redundant given n_categories. When one tree and the forest disagree, trust the average of 100 trees. Final 8 features validated by 5-fold CV: accuracy 0.885, precision 0.746, recall 0.582, F1 0.652.

4 · Dimensionality reduction — PCA & SVD

Selection picks features; PCA/SVD create compressed ones (PCA on scaled features ≈ SVD of the covariance matrix).

Fig A · PCA elbow. Cumulative explained variance vs number of components: 2 components capture ≈58%, 3 ≈70%. The curve justifies keeping few components for visualization.
Fig B · PCA churn map. The 295 researchers projected to 2D, colored by churn. Red (churned) separates from green (active) along PC1 (recency) — visible separation is the proof the engineered features carry real signal.
Fig C · SVD of the researcher×category matrix (scipy.sparse.linalg.svds, k=12, 84% variance). Left: scree — the first factor alone holds ~70% of the structure. Right: authors (circles, by churn) and candidate papers (gold) in the recommender's latent space; proximity = shared research interest, which is exactly how recommendations are chosen.
PCA components as model inputs underperform. Training on 3 PCA components costs 0.115 F1 (0.537 vs 0.652) and drops recall to 0.473: PCA compresses for variance, not churn discrimination. It earns its place as explanation (Figs A–B), not as the production feature set.

5 · Network analysis as feature enrichment

A co-authorship graph (networkx; edge = shared pre-cutoff paper) yields degree, betweenness and PageRank centralities, added as features and re-run through all four selection methods.

Fig D · Co-authorship network. 295 researchers, 417 edges (node size = degree). Sparse, low-degree nodes on the rim skew red: isolated researchers churn more — collaboration is retention.
Fig E · RandomForest importance. Recency (33%) and recent-output share (19%) dominate the model — the quantitative echo of the four selection methods and of the unit's motto.
Effect. No single centrality wins the consensus vote alone (their signal emerges jointly with temporal features), yet together they lift F1 from 0.652 to 0.678 and recall to 0.618.

7 · Acting on the prediction — recommendation, ethics, PM view

Recommendation. The researcher×category matrix is factorized with truncated SVD; POST /recommend returns the most cosine-similar recent papers by active authors — but only when churn probability ≥ 0.5 (engaged users get a “not at risk” reply). Re-engagement budget belongs on the at-risk segment.

Ethics. A churn score is a probability, not a verdict: interventions are designed to be harmless on false positives (a relevant-papers digest), and the model must never feed hiring/funding decisions — leaving arXiv ≠ leaving science. The label is an honest proxy (silence), surfaced by /features; author names are string-matched, so homonyms are a disclosed limitation.

PM reflection. As a product manager I would run this as one predict → explain → act loop: predict nightly to produce a ranked re-engagement budget; let the feature-selection table route the channel (recency-risk → content nudges, isolation-risk → collaboration features); and treat acting as an A/B experiment that measures incremental retention and feeds the threshold back. The transferable lesson: churn is rarely sudden — cadence decays first, so watching the derivative of engagement lets you act while there is still something to save.
